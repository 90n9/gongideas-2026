ผมเพิ่งไปเจอข้อจำกัดนี้ตอนที่พยายาม automate การโพสต์บทความจาก GongIdeas ลง Facebook page ครับ

## ปัญหา: เพจ Facebook ฟรีโพสต์ลิงก์ได้แค่ 4 ครั้งต่อเดือน

Facebook มีนโยบายจำกัดการโพสต์ที่มี **external link** สำหรับ page ที่ยังไม่ verified — โพสต์ได้แค่ **4 ครั้งต่อเดือน** เท่านั้น ถ้าเกินโควต้า โพสต์จะไม่ขึ้น error ให้เห็น แต่จะ **ถูก drop เงียบ ๆ** — ไม่มีแจ้งเตือน ไม่มีข้อความอะไรเลย

ซึ่งทำให้ debug ยากมากครับ เพราะ UI บอกว่าโพสต์สำเร็จ แต่โพสต์ไม่ขึ้นในหน้า feed

## วิธีแก้: โพสต์ content ก่อน แล้วค่อยเพิ่มลิงก์ใน comment

วิธีที่ใช้ได้ผลคือแยกลิงก์ออกจาก post body:

1. **โพสต์ content ปกติ** — ใส่ข้อความ บทความ หรือ highlight ทั้งหมดลงใน post body โดยไม่มี URL
2. **เพิ่มลิงก์เป็น comment แรก** — หลังโพสต์สำเร็จ ค่อยคอมเมนต์แนบลิงก์ไว้ใต้โพสต์

Facebook นับแค่ลิงก์ที่อยู่ใน **post body** ครับ ลิงก์ใน comment ไม่นับโควต้า

## ทำไมเพจดัง ๆ ถึงชอบพูดว่า "ลิงก์อยู่ในคอมเมนต์"

เคยสงสัยไหมครับว่าทำไมหลายเพจ โดยเฉพาะเพจข่าว เพจ content creator หรือเพจ media ต่าง ๆ ถึงชอบโพสต์ว่า "ดูลิงก์ได้ในคอมเมนต์" หรือ "link in comments" แทนที่จะแปะลิงก์ตรง ๆ ในโพสต์?

คำตอบคือ **นี่คือเหตุผลหลัก** — ไม่ใช่แค่สไตล์การโพสต์หรือความชอบส่วนตัว แต่เป็นเพราะโควต้าลิงก์ของ Facebook page นั่นเอง เพจที่โพสต์บ่อย ๆ หลายครั้งต่อเดือนไม่มีทางรอดถ้าแปะลิงก์ในทุกโพสต์

นอกจากนี้ยังมีเหตุผลรองด้วยครับ — Facebook algorithm มักลดการ reach ของโพสต์ที่มีลิงก์ออกนอกแพลตฟอร์ม เพราะ Facebook ไม่อยากให้คนออกจาก platform การวางลิงก์ไว้ใน comment จึงช่วยให้ algorithm กระจาย reach ได้กว้างกว่าด้วย

## สรุป

- **โควต้า 4 ลิงก์ต่อเดือน** คือข้อจำกัดจริงของ Facebook page ฟรี
- **วิธีแก้**: โพสต์ content ใน post body, ใส่ลิงก์เป็น comment แรก
- **"ลิงก์ในคอมเมนต์"** ที่เห็นในเพจดัง ๆ มาจากข้อจำกัดนี้แทบทั้งนั้น ไม่ใช่แค่ style

ถ้ากำลัง automate การโพสต์ลง Facebook page หรือวางแผน content calendar — รู้ไว้ก่อนจะได้ไม่งงครับ
